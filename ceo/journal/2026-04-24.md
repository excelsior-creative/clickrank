## Inspection findings ‚Äî the design redesign reintroduced every
fabrication we killed on 2026-04-19

system") was merged 2026-04-19. It replaced the homepage, `/about`,

  test), CTA "How we test", featured card with a fabricated score
  of 8.4 in the default and a real fabrication on live posts.

  3.5..9.2`. Every "Editor's Pick" rating on the homepage was a hash
  of the slug.
- **`PostCard.tsx`** ‚Äî same hash-of-slug score rendered on every card
  in the review grid, everywhere PostCard appears. Plus byline
  "By the ClickRank desk" on every card.
- **`ProcessSection.tsx`** ‚Äî six-step "how we test" with claims of
  "Hands-on testing" ("we buy the product, work through it"),
  "Quality assurance" ("a second editor reads the draft with the

  tell us‚Ä¶"), all false.
- **`CommitmentSection.tsx`** ‚Äî "A small team of editors, each with a
  real name and a real beat, spends an average of 14 hours on every

- **`ReviewsSection.tsx`** ‚Äî subhead "Every review below was **tested

- **`CTABanner.tsx`** ‚Äî newsletter subscribe form that set local state

## Changes & risk

- All changes are additive or subtractive on render-layer components.
  No DB writes, no migrations, no auth or cron changes.
- `TrustRow` now makes two DB `count()` calls + one single-row `find`
  at request time. Homepage already hits the DB for featured post
  and the review grid; this adds ~1ms per render on a warm DB, and
  the page is ISR 60s so the hit rate is tiny.
- `PostCard` no longer shows a score for existing posts because
  `rating` isn't in the schema. That's the correct state ‚Äî rendering
  a fabricated score was worse than rendering none. When the
  migration lands, real scores will start appearing card-by-card as
  editorial fills them in.
- `CTABanner` is now a server component (no "use client"). It had
  `useState` for the fake subscribe flow; removed along with the
  form. No other side effects depended on it.
- `Footer` default link list changed. If the CMS footer has
  configured nav items, those still take priority ‚Äî the defaults
  only render when the CMS global is empty.
- Type-check: `pnpm --filter app type-check` passes.
