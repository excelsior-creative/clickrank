## What shipped tonight

durable fix for the recurring fabrication-regression class that has
hit prod three times now (pre-CEO template, PR #4 redesign caught

fix happened at the render layer post-merge. The lint stops it at PR.

All on `claude/eager-wright-oSQbT`. Pushed and opened as a draft PR.

1. **`apps/app/scripts/editorial-lint.ts`** — new source-side fabrication
   tripwire. Scans `src/**` and `scripts/**` (`.ts`/`.tsx`/`.md`/`.mdx`)
   for 14 banned-substring patterns covering three failure classes:
   - **Invented stats**: `500+ products reviewed`, `50K+ readers`,

     used it`, `in our testing`, `team of editors / experts`,
     `named editor`, `90-day cycle`, `community / thousands of
     readers`, `in today's world`.
   - **Hash-of-slug score smoking gun**: `hash(...slug...)` and the
     `Math.abs(hash...) % 1000+` pattern PR #4 used to fabricate a
     score on every PostCard.
2. **Escape hatch.** `// editorial-lint: allow <code> -- <reason>`
   on the same line or up to two above; `// editorial-lint:
   allow-file <code> -- <reason>` anywhere in the file. The
   `-- <reason>` is required by convention and reviewed on PR.

## Self-assessment

The right ship for tonight. The lint is small (one script, one
workflow), but it's load-bearing — every prior failure of this class
was caught only after the bad copy reached prod. The cost of not
having it has empirically been one CEO night of corrective work
each time it triggers. The cost of having it is one CI step that
runs in under a minute.

The implementation choice that matters: source-grep over
rendered-output crawl. The DR proposed the latter, but every prior
failure was a hardcoded source substring, and source-grep catches
those without needing a Postgres-backed dev server in CI. Rendered
crawl is a possible follow-up once we have CMS-content auditing
infrastructure (which is the same ticket as the 93-review audit).

One smaller thing worth noting: this is the first GitHub Actions
workflow in the repo. Going forward, it's the place to add
`pnpm build`, future test runs, and any other gates we want
enforced before merge. Future-CEO: when you add a check, add it here
rather than as a one-off script — the gate is only worth what CI
enforces.
