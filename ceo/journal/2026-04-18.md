## What shipped tonight

1. **Pipeline: editorial QA gate** (new file
   `apps/app/src/services/qaService.ts`, wired into
   `/api/cron/generate-article`). First load-bearing safety gate on
   the pipeline. Checks:
   - Required fields present (title, slug, excerpt, content).
   - Slug is valid kebab-case.
   - `## Affiliate disclosure` heading + commission/earn language
     present in body (FTC compliance).
   - Affiliate or vendor link present in body (warning if missing).
   - "What could be better"-style caveats section present (honest-
     favorable standard; warning).
   - Word count in [700, 3500] (<700 is error, >3500 is warning).
   - ≥4 headings (warning).
   - Title ≤80 chars, meta description ≤170 chars (warnings).
   - Fabrication tells: first-person testing claims, weight-loss
     claims, fake user-count claims, oddly specific stats — errors.
   - Forbidden claims: guaranteed income/results, FDA-approved,
     miracle cures — errors. Clinically/scientifically proven,
     guaranteed weight loss — warnings.
   - Em-dashes remaining after humanization — warning.
   - Errors block persistence (422 response). Warnings are logged
     and included in the JSON response.

2. **Pipeline: dedup fix.** `filterUnusedProducts` was using
   `slug.includes(productSlug.slice(0, 15))` which false-positives on
   any slug containing the same 15-char prefix. Replaced with a
   canonical-token intersection: product name is tokenized, stopwords
   and year markers stripped, and we match only if ALL significant
   tokens appear in a recent slug. Prevents "Ted's Woodworking" from
   colliding with an unrelated "ted-talks-review" type slug.

3. **Site: Footer cleanup.**
   - Removed the decorative newsletter form (input + button with no
     submit handler). Actively misleading; the backlog flagged it.
   - Removed the four social-media `href="#"` icons. No account =
     no link.
   - Footer now reads: logo + tagline, nav, legal, affiliate
     disclosure, copyright. Simpler and honest.
   - Copyright line now reads from `siteSettings.siteTitle` instead
     of hardcoded "ClickRank.NET".

4. **Site: `/blog` index cleanup.**
   - Switched from `dynamic = 'force-dynamic'` to ISR with
     `revalidate = 3600`. The Posts `afterChange` hook already does
     on-demand revalidation on publish, so hour-level fallback is
     plenty. Should measurably improve TTFB on the index.
   - Subtitle copy updated from template-leftover "Insights, updates,
     and tutorials from our team about modern web development and
     design" to ClickRank-voice: "Honest, independent reviews of
     digital products from the ClickBank marketplace. Leading with
     real strengths, naming real caveats, and keeping affiliate
     relationships transparent."
   - Badge + title updated to "Reviews" / "All reviews".

5. **Site: `/blog/[slug]` cleanup.** Same ISR switch as the index.
   Posts hook revalidates on change; 1h safety net otherwise.

## Changes & risk

- QA service is pure functions, no network, no state. If it
  accidentally rejects every single generated draft, the worst case
  is zero new posts for a night — we fail closed, which is the right
  direction.
- Dedup change can't crash the pipeline (falls back to the full
  product list if nothing survives). If it loosens too much and we
  start repeating products, Brandon will see it in `products.md`
  tomorrow night.
- Footer removals are purely deletions of unused UI — no regression
  risk.
- ISR changes: on-demand revalidation via the Posts hook is the
  primary freshness source; 1h fallback. If the revalidate hook
  fails silently we'd have up-to-1h stale index, which is acceptable
  and not worse than the current "force-dynamic" (which also serves
  stale between requests because of Next's route caching quirks).
