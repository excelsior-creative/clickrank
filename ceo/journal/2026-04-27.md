## ESLint circular-config fix

- Investigated Brandon's request to fix the lint circular-config issue. Root cause: `apps/app/eslint.config.mjs` used `FlatCompat.extends("next/core-web-vitals", "next/typescript")`, but `eslint-config-next@16` already exports flat config arrays. Passing those through legacy compat triggered `@eslint/eslintrc` schema validation and attempted to stringify plugin objects with circular references (`plugins.react.configs.flat...plugins.react`).
- Replaced the compat wrapper with a direct flat-config import from `eslint-config-next/core-web-vitals`.
- After the circular issue was removed, lint exposed normal file-level errors. Fixed the blocking errors by using `next/link` for internal category links, escaping JSX copy, and documenting the intentional next-themes mounted-state effect suppression.
